# 长期目标关联（Long-term Goal Alignment）功能设计文档（修改版）

## 1. 功能定位

长期目标关联功能不是单独替代原有决策流程，而是在原有“记录决策 → AI 建议 → 后续复盘 → 结果分析”的基础上，增加一个长期维度。

本功能采用 **“独立目标模块 + 全局决策关联”** 的设计：

- **首页 / 决策中心**：仍然负责日常决策记录、待回看、已复盘和满意度统计。
- **长期目标页面**：独立负责目标管理、目标详情、目标进展和目标相关决策分析。
- **新建 / 编辑决策**：可选择关联一个长期目标，非必填。
- **目标详情页**：可以直接在某个目标下创建决策，并自动关联该目标。
- **AI 建议**：当决策关联目标时，AI 需要额外分析每个候选方案与长期目标的契合度。

这样既避免长期目标功能变成孤立模块，也避免把首页做得过于复杂。

---

## 2. 业务背景与价值

在原有系统中，用户记录的是一次次独立决策。例如：

- 是否报名周末课程
- 是否接受兼职
- 是否减少咖啡摄入
- 是否参加线下活动

这些决策本身有价值，但它们之间缺少长期联系。引入“长期目标”后，系统可以帮助用户判断：

> 这次选择是否让自己更接近长期目标？

例如用户设置长期目标：

> 一年内提升编程能力，找到开发实习。

当用户创建决策“是否报名周末前端课程”时，系统可以结合该目标，分析不同候选方案是否有利于目标达成。

本功能的核心价值是：

1. 让用户的单次决策不再孤立。
2. 帮助用户从长期角度判断当前选择。
3. 为 AI 建议增加更有个性化和长期性的依据。
4. 让复盘结果不只停留在“满意 / 一般 / 后悔”，还可以进一步分析“是否符合长期目标”。
5. 提升“决策回声”系统的产品特色，使其从决策记录工具升级为长期决策辅助系统。

## 3. 功能结构设计

### 3.1 整体关系

```text
长期目标 Goal
   ↓ 关联
决策 Decision
   ↓ 生成
AI 建议 Advice
   ↓ 后续
复盘 Review
   ↓ 汇总
结果分析 / 目标进展分析
```

### 3.2 推荐页面结构

侧边栏菜单建议为：

```text
首页
统计
长期目标
我的
```

其中：

- **首页**：展示所有决策记录、待回看任务、满意度概览。
- **统计**：展示整体复盘结果分析。
- **长期目标**：展示目标列表、目标详情、目标相关决策、目标契合度分析。
- **我的**：用户信息、账号设置等。

## 4. 核心功能实现清单

## 4.1 目标管理

### 4.1.1 新建目标

用户可以创建长期目标，填写：

- 目标标题
- 目标描述
- 目标分类
- 目标优先级
- 预期完成日期
- 衡量方式
- 当前进度
- 当前状态

示例：

```text
目标标题：提升编程能力，找到开发实习
目标分类：学习 / 工作
目标优先级：高
衡量方式：完成 3 个项目，并投递 30 个岗位
预期完成日期：2026-12-31
当前进度：40%
状态：进行中
```

### 4.1.2 查看目标列表

在“长期目标”页面中，以卡片形式展示所有目标。

每个目标卡片建议展示：

- 目标标题
- 目标状态
- 目标优先级
- 目标分类
- 当前进度
- 预期完成日期
- 距离截止日期的剩余时间
- 已关联的决策数量

### 4.1.3 修改 / 删除目标

支持用户修改目标信息。

删除目标时，建议采用以下策略：

- 删除目标后，已关联的决策不删除。
- 对应决策的 `goal_id` 设置为 `NULL`。
- 前端删除前需要二次确认。

### 4.1.4 目标状态

目标状态建议包括：

```text
IN_PROGRESS：进行中
COMPLETED：已完成
ABANDONED：已放弃
```

第一版可以先只支持这三个状态。

## 4.2 决策与目标关联

### 4.2.1 首页创建决策时关联目标

在原有“新建决策”表单中增加一个可选字段：

```text
关联长期目标：请选择，可不选
```

只展示状态为 `IN_PROGRESS` 的目标。

该字段非必填，用户仍然可以创建普通决策。

### 4.2.2 目标详情页创建决策

在目标详情页中提供按钮：

```text
+ 为此目标创建决策
```

点击后打开新建决策弹窗，并自动选中当前目标。

这样用户可以在查看目标时直接记录相关决策，降低关联成本。

### 4.2.3 决策详情页展示关联目标

如果某条决策关联了长期目标，决策详情页需要展示：

```text
关联目标：提升编程能力，找到开发实习
```

点击目标名称可以跳转到目标详情页。

### 4.2.4 决策卡片展示关联目标

首页决策卡片中可以轻量展示目标标签：

```text
目标：提升编程能力
```

建议以弱化的小标签形式展示，不要抢占标题和最终选择的视觉层级。

## 4.4 AI 智能评估增强

### 4.4.1 Prompt 注入逻辑

当决策关联了目标时，后端在调用 AI 前，需要查询该目标信息，并追加到 Prompt 中。

Prompt 示例：

```text
【用户的长期目标】
标题：{goal.title}
描述：{goal.description}
分类：{goal.category}
优先级：{goal.priority}
衡量方式：{goal.measurement}
预期完成日期：{goal.targetDate}
当前进度：{goal.progress}%

请在分析以下候选方案时，额外评估每个选项是否有助于实现该长期目标。
请输出“长期目标契合度分析”，并指出哪个选项最有利于该目标达成。
```

### 4.4.2 AI 建议内容增加目标维度

原有 AI 建议内容包括：

- 利弊分析
- 风险提醒
- 推荐选项
- 总结建议

新增：

- 长期目标契合度分析
- 每个候选方案的目标契合度评分
- 最有利于长期目标的选项
- 可能偏离目标的风险提醒

### 4.4.3 建议使用结构化返回

为了方便前端展示和后续统计，建议 AI 返回结构化 JSON。

示例：

```json
{
  "summary": "总体建议内容",
  "recommendedOption": "报名课程",
  "goalAlignment": {
    "score": 85,
    "level": "高度契合",
    "bestOption": "报名课程",
    "reason": "该选项能够提升前端能力，并为长期求职目标积累项目经验。",
    "optionAnalysis": [
      {
        "option": "报名课程",
        "score": 85,
        "comment": "直接提升技能，符合长期目标。"
      },
      {
        "option": "不报名",
        "score": 40,
        "comment": "短期节省时间，但对长期目标帮助较小。"
      }
    ]
  }
}
```

### 4.4.4 AI 建议页展示方式

AI 建议页新增独立模块：

```text
长期目标契合度
契合度：85%
推荐选项：报名课程
原因：该选择能直接提升前端能力，并为转行目标积累项目经验。
```

这个模块可以放在原有建议结论之后，或者放在建议卡片右侧。

## 5. 技术实现设计

## 5.1 数据库设计

### 5.1.1 新增 `goal` 表

```sql
CREATE TABLE goal (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '目标ID',
    user_id BIGINT NOT NULL COMMENT '所属用户ID',
    title VARCHAR(100) NOT NULL COMMENT '目标标题',
    description TEXT COMMENT '目标详细描述',
    category VARCHAR(50) DEFAULT NULL COMMENT '目标分类：学习、工作、健康、财务等',
    priority VARCHAR(20) DEFAULT 'MEDIUM' COMMENT '优先级：HIGH, MEDIUM, LOW',
    status VARCHAR(20) DEFAULT 'IN_PROGRESS' COMMENT '状态：IN_PROGRESS, COMPLETED, ABANDONED',
    target_date DATE COMMENT '预期达成日期',
    measurement VARCHAR(255) DEFAULT NULL COMMENT '目标衡量方式',
    progress INT DEFAULT 0 COMMENT '目标进度，0-100',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_user_id (user_id),
    INDEX idx_status (status),
    INDEX idx_target_date (target_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户长期目标表';
```

### 5.1.2 修改 `decision` 表

第一版采用一个决策关联一个目标的方案，简单稳定。

```sql
ALTER TABLE decision ADD COLUMN goal_id BIGINT DEFAULT NULL COMMENT '关联的长期目标ID';

ALTER TABLE decision
ADD CONSTRAINT fk_decision_goal
FOREIGN KEY (goal_id) REFERENCES goal(id)
ON DELETE SET NULL;
```

说明：

- `goal_id` 可以为空。
- 删除目标时，不删除决策，只解除关联。
- 第一版先使用一对多关系：一个目标可以关联多个决策，一个决策最多关联一个目标。

### 5.1.3 后续可扩展为多对多

未来如果需要支持一个决策关联多个目标，可以新增中间表：

```sql
CREATE TABLE decision_goal (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    decision_id BIGINT NOT NULL,
    goal_id BIGINT NOT NULL,
    alignment_score INT DEFAULT NULL COMMENT '目标契合度评分',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_decision_goal (decision_id, goal_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='决策与目标关联表';
```

但第一版暂不建议直接做多对多，避免增加实现复杂度。

## 5.4 AI 服务改造

修改 `AdviceServiceImpl.buildCreatePrompt` 方法。

处理流程：

```text
1. 接收 Decision 对象。
2. 判断 decision.goalId 是否为空。
3. 如果不为空，调用 GoalService 查询目标详情。
4. 校验该目标是否属于当前用户。
5. 将目标信息追加到 AI Prompt。
6. 要求 AI 输出长期目标契合度分析。
7. 将 AI 返回结果解析并保存。
```

### 5.4.1 建议新增响应字段

如果当前已有 `DecisionAdviceResponse`，建议增加：

```java
private GoalAlignmentDTO goalAlignment;
```

`GoalAlignmentDTO` 示例：

```java
public class GoalAlignmentDTO {
    private Integer score;
    private String level;
    private String bestOption;
    private String reason;
    private List<OptionAlignmentDTO> optionAnalysis;
}
```

`OptionAlignmentDTO` 示例：

```java
public class OptionAlignmentDTO {
    private String option;
    private Integer score;
    private String comment;
}
```

## 7. 首页与长期目标页面的关系

本功能采用“联系起来，但不混在一起”的设计。

### 7.1 首页职责

首页继续承担：

- 查看决策列表
- 新建决策
- 查看待回看任务
- 查看已复盘记录
- 查看满意率概览

首页不直接展示完整长期目标管理功能，只做轻量关联。

### 7.2 长期目标页面职责

长期目标页面承担：

- 创建目标
- 管理目标
- 查看目标详情
- 查看目标下的决策
- 查看目标进展
- 从目标下创建决策

### 7.3 两者如何联系

```text
首页新建决策 → 可选择关联长期目标
目标详情页新建决策 → 自动关联当前目标
决策详情页 → 展示关联目标并可跳转
目标详情页 → 展示所有关联决策
AI 建议页 → 展示目标契合度分析
```

## 8. MVP 实现范围

第一版建议实现以下内容：

```text
1. 新增长期目标菜单
2. 目标列表页
3. 新建 / 编辑 / 删除目标
4. 目标详情页
5. 决策创建 / 编辑时可选择一个进行中的目标
6. 目标详情页可以创建自动关联当前目标的决策
7. 决策卡片和详情页展示关联目标
8. AI 建议中增加长期目标契合度分析
9. 目标详情页展示关联决策列表
```

第一版暂不建议实现：

```text
1. 一个决策关联多个目标
2. AI 自动推荐关联目标
3. 复杂目标趋势图
4. 自动计算目标进度
5. 复杂目标一致率分析
```

这些可以放到第二版。

## 9. 第二版可扩展功能

后续可以考虑增加：

### 9.1 AI 自动推荐关联目标

用户输入决策标题和背景后，AI 自动判断可能相关的目标。

示例：

```text
可能关联目标：提升编程能力，找到开发实习
```

### 9.2 多目标关联

一个决策可以同时关联多个目标。

适合场景：

```text
是否接一个兼职
```

可能同时影响：

- 财务目标
- 学习目标
- 健康目标

### 9.3 目标一致率分析

系统统计某个目标下的历史决策中，有多少是有利于该目标的。

示例：

```text
目标一致率：75%
最近 30 天有 6 个决策与该目标高度契合。
```

### 9.4 目标复盘报告

系统每周或每月生成目标复盘报告。

示例：

```text
本月你围绕“提升编程能力”的目标做了 5 个决策，其中 4 个有助于目标推进。
建议下个月减少低优先级活动，把更多时间投入项目实践。
```

## 12. 总结

长期目标关联功能的最佳设计不是完全独立，也不是完全塞进首页，而是：

```text
长期目标独立成模块，决策流程全局可关联目标。
```

最终形成的产品逻辑是：

```text
首页负责记录和复盘单次决策；
长期目标页负责观察这些决策是否长期一致；
AI 建议负责判断当前选择是否有助于长期目标。
```

这样既保持首页简洁，又能让长期目标功能真正融入用户的日常决策流程。
